The Easiest Way To Understand Git

Git is a popular software among the developer community. It enables them to manage and track code changes as well as reverting to various code status. It also helps simultaneous team collaboration on projects. In this short blog, you will learn the basics of Git

What is Git?

Git is a version control system that is used to manage and keep track of files and code changes over time.

A version control system is a software that is used to manage changes to files over time, keep track of every modifications made, enabling users to go back to various states of their files.

There are several version control systems currently used in the world, they include but not limited to Mercurial, Azure DevOps, GNU Bazaar, and Git.

In this blog, we will learn how to setup and use Git, one of the most used version control systems today.

Installing Git

After having an understanding of what Git is, lets go through the steps to having it installed in you machine.

  1. Confirm whether Git is already installed in your coputer.
    2.

    Some operating systems come with Git installed, so it's best to always confirm before installing. To confirm, run this command on your terminal:

    git --version

    If the output is a Git version such as git version 2.34.1, it means that Git is already installed, otherwise head on to the next step.

  2. Download Git for the main website.
    3.

    To install Git, head on to the git download webpage:

    Choose your operating system and follow the downloading steps. Note: the downloading procedures differ across various operating systems.

  3. Confirm the installation.
    4.

    To confirm whether the installation was successful, run this command on the terminal:

    git --version

    The expected output, is the version of git installed e.g git version 2.34.1.

    If there are any problems, refer to the git main website

Configuring Git

Now that you have Git installed, you should let Git know who you are by configuring your name and the email address.

To add your name, run the following command on the terminal:

git config --global user.name "Your Name"

To add your email address, run the following command on the terminal:

git config --global user.email "Your Email Address"

Working with Git Repositories

A Git repository is a storage location where all changes will be managed and tracked.

Initilizing a repository

To initialize a Git repository, you should first Create a new folder for the project on your Desktop, (You can use any directory of your choice), by using the following command on the terminal:

mkdir "Project Name" ; cd "Project

The initialize Git by running the command:

git init

Adding files to the staging area

After making changes to your new project, use the following command to check the status of your created files:

git status

When a file is Untracked it means that Git knows about the file but it has not been added to the repository, while when a file is Tracked it means that it has been added to the git repository

For a file to be tracked, it has to be added to the staging area.

A staging area is basically a space where chages to a file are prepared before they are commited to a repository.

To add a file to the staging area use the following git command on the terminal:

git add "filename"

To add all the files, use the command:

git add . Or git add all

Commiting changes

A commit is basically a chunk of changes made to a repository at a particular point in time.

To commit changes, use the following command on the terminal:

git commit -m "Commit Message"

The commit message should always be descriptive, explaining what the commit will do. A good practice is to ask yourself the question,"This commit will?" For example, "This commit will intialize the project," initialize the project will be the commit message.

To show a list all commits made, use the following command on the terminal:

git log --oneline

To go back to a particular commit, use the following command on the terminal:

git checkout "commit code"

Branches

A branch in Git is a separate copy of the main repository used to allow multiple users to work simultaneously on the same project without interfering with each other's woek.

Creating a branch

To create a branch, use the following command on the terminal:

git branch "branch name"

Viewing all branches

To view the branch present in a repository, use the following command on the terminal:

git branch

The branch with an asteric prefixed, is the active branch

Switching active branches

To change branches, or rather to switch to a different branch, use the following command on the termina:

git checkout "branch name"

The branch name, is the name of the branch you want to switch to.

Merging a branch

To merge a branch is to add changes from one branch branch to another, mostly to the ain branch.

To merge changes, switch to that branch and use the following command on the terminal.

git merge "branch name"

The branch name, is the name of the branch from which changes as merged.

Deleting a branch

To delete a branch, use the following command on the terminal:

git branch -d "branch name"

Additional git commands

  • git branch -M "newBranchName" for renaming a branch

  • git log for listing all the commits in details

  • git switch "branchName" for switching between branches
